<section class="home-section">
  <h2>The Point of View</h2>
  <div class="home-grid">
    <article>
      <h3>Keep what works</h3>
      <p>If an existing platform is valuable, make it cleaner, faster, and easier to operate. Use what you already purchased well.</p>
    </article>
    <article>
      <h3>Sunset what doesn’t</h3>
      <p>When a system is expensive, slow to change, and no longer matched to the team, replace the smallest useful part first.</p>
    </article>
    <article>
      <h3>Build for the actual workflow</h3>
      <p>Purpose-built tools can match the way people work instead of forcing people to work the way a platform wants.</p>
    </article>

<section class="home-section">
  <h2>How I Use AI</h2>
  <div class="home-grid">
    <article>
      <h3>Claude for the plan</h3>
      <p>I use Claude while the problem is still fuzzy: ideating, pressure-testing assumptions, mapping options, and working through decisions before code exists. A plan only matters if it turns into something real.</p>
    </article>
    <article>
      <h3>Codex for the grind</h3>
      <p>I use Codex once the direction is clear: implementing, refactoring, writing tests, wiring commands, reviewing diffs, and pushing through details. It works best with a tight scope and a way to prove the change.</p>
    </article>
    <article>
      <h3>OpenClaw and local AI for control</h3>
      <p>I use local tools for private exploration, sensitive workflows, offline review, and helping people learn what is possible without handing every task to a cloud service. Production paths still need reliability and traceability.</p>
    </article>
  </div>
</section>

<section class="home-section">
  <h2>What This Looks Like</h2>
  <div class="home-grid">
    <article>
      <h3>Legacy platform audit</h3>
      <p>Separate useful infrastructure from inherited complexity, unused licenses, duplicate workflows, and avoidable integration cost.</p>
    </article>
    <article>
      <h3>Purpose-built replacement</h3>
      <p>Design and ship a smaller tool that solves the real job: intake, review, tracking, approvals, reporting, handoffs, or customer operations.</p>
    </article>
    <article>
      <h3>Fast iteration loop</h3>
      <p>Use Claude for planning, Codex for implementation, OpenClaw and local AI for private exploration, and tight feedback cycles to improve the tool while the team is already using it.</p>
    </article>
  </div>
</section>
